
```json
{
  "{{配置项1}}": "{{配置值}}",
  "{{配置项2}}": "{{配置值}}",

```

```bash

```

### 版本号规则

| 版本格式 | 说明 | 示例 |
|----------|------|------|
| vX.Y | Y=0 为大版本，Y>0 为小版本 | v1.0, v1.1, v2.0 |

### 开发流程

```
┌─────────────────────────────────────────────────────────────┐
│  用户提出需求/反馈Bug                                        │
│  记录沟通轮次: 第X次                                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  1. [沟通记录.md](沟通记录.md) - 记录讨论过程                  │
│     记录：第X次、YYYY-MM-DD HH:mm、讨论要点                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. [PRD.md](PRD.md) - 更新需求文档                          │
│     更新：版本号 → vX.Y、变更说明                            │
│     标记：PRD.md 开头写明版本变化                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. [开发记录.md](开发记录.md) - 列出任务 + 方案              │
│     更新：版本号、待办任务、CSS/代码方案                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
                        用户确认文档 ✅
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  ★ 确认需求/Bug清单（强制）                                  │
│     用户说"执行开发流程"后，必须先展示：                       │
│     - 本次所有需求（表格形式）                                │
│     - 本次所有Bug（表格形式）                                │
│     - 询问用户确认是否有遗漏                                 │
│     - 用户确认后再次提示"是否按开发流程执行？"                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 执行代码修改                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. [变更日志.md](变更日志.md) - 记录本次变更                 │
│     更新：vX.Y 内容、新增/修改/修复项                        │
│     [开发记录.md](开发记录.md) - 标记任务完成                 │
└─────────────────────────────────────────────────────────────┘
```

**原则**：先改文档，用户确认后再写代码。
**强制**：执行开发流程前必须先确认需求/Bug清单。
