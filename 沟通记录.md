### 需求：AI气泡支持Markdown渲染

#### 用户需求
- AI答复的气泡区域，希望支持最简单的md内容显示
- 不用太复杂，轻量一点

#### 技术方案确认

**方案选择：引入第三方库 marked.js**

| 方案 | 复杂度 | 依赖 |
|------|--------|------|
| 手写正则解析 | 低，约30行代码 | 无 |
| 引入 marked.js 库 | 中，需要引入库 | CDN加载 |

**最终选择：marked.js**
- 轻量级库（约45KB）
- 支持完整的Markdown语法
- CDN加载，无需本地安装

#### 支持的Markdown语法
| 语法 | 效果 |
|------|------|
| `**粗体**` | **粗体** |
| `*斜体*` | *斜体* |
| `` `代码` `` | `代码` |
| ``` ```代码块``` ``` | 代码块（多行） |
| `[链接](url)` | 可点击链接 |
| `- 列表` | 无序列表 |
| `> 引用` | 引用块 |
| `# 标题` | 标题 |

#### 修改文件清单
| 文件 | 修改内容 |
|------|---------|
| `public/index.html` | 引入 marked.js CDN |
| `public/js/chat.js` | AI消息使用 `marked.parse()` 渲染 |
| `public/css/style.css` | 添加 Markdown 样式 |

## 2026-05-06 沟通记录（09）

### Bug 1：缩略图文件名乱码

**问题描述：**
- 视频 ID 包含中文（如 `video_油罐车的构造原理是什么...moerk303`）
- 保存为文件名时出现编码乱码

**确认方案：**
- 使用 MD5 哈希生成 16 位安全文件名
- 例如：`video_油罐车...moerk303.jpg` → `a1b2c3d4e5f6g7h8.jpg`

### Bug 2：手机端标题显示3行

**问题描述：**
- 宽屏显示正常（2行），但手机端仍显示3行
- `-webkit-line-clamp: 2` 在小屏幕字体下不生效

**确认方案：**
- 添加固定高度限制 `height: 36px`
- 配合 `overflow: hidden` 强制截断

### Bug 3：Welcome 区域手机端不水平居中

#### 问题描述
- Welcome 区域（欢迎页）在手机屏幕上没有水平居中
- 整体靠左显示

#### 排查过程（4轮）

| 轮次 | 尝试方案 | 问题 | 结果 |
|------|----------|------|------|
| 1 | `.welcome-screen` 添加 `width: 100%` | 误以为是 flex 容器需要宽度 | ❌ 未生效 |
| 2 | 添加 `max-width: 600px` + `margin: 0 auto` | 方向正确但未理解本质 | ❌ 未生效 |
| 3 | 修改父容器 `.chat-container` 添加 `display: flex; align-items: center` | **错误理解！**导致所有内容包括气泡都居中了 | ❌ 大错特错 |
| 4 | 恢复到 `.welcome-screen`，改为 `text-align: center` | 理解问题本质：只需要文字居中 | ✅ 成功 |

#### 问题根因
用户说的是 **welcome 区域水平居中**，但实际上需要的是 **welcome 区域内的文字/内容居中显示**。
- 之前一直尝试让 `.welcome-screen` 容器本身居中
- 但父容器 `.chat-container` 有 `padding: 16px`，容器本身就是贴边的
- 真正需要的是让 welcome 内的内容水平居中

#### 最终解决方案
```css
.welcome-screen {
    text-align: center;  /* 核心：让内部内容水平居中 */
    padding-top: 40px;
    padding-bottom: 20px;
}
```

#### 经验教训
1. **理解需求要准确**：用户说"区域居中"不一定是容器居中，可能是内容居中
2. **避免过度修改**：第3轮修改了父容器，导致副作用（气泡也居中了）
3. **从问题本质出发**：welcome 区域本质是文字块，`text-align: center` 才是正确方案

### 需求：视频卡片缩略图优化

#### 原始问题
- 视频卡片只显示 `🎬` emoji，无真实缩略图
- 标题显示3行，不够简洁

#### 讨论方案

| 方案 | 描述 | 可行性 |
|------|------|--------|
| 前端 video+canvas | 浏览器截取首帧 | 需要解决 video 元素复用问题 |
| FFmpeg | 服务端命令行处理 | 太重，需要安装额外软件 |
| Python OpenCV | 服务端预处理 | 轻量，推荐方案 |

#### 确认方案：Python OpenCV 预处理

**流程：**
1. 服务端启动时运行 `generate_thumbnails.py`
2. 检测 videos.json 中没有 thumbnail 的视频
3. 用 OpenCV 读取视频第 0.5 秒帧
4. 保存为 JPG 到 `public/thumbnails/`
5. 更新 videos.json 的 thumbnail 字段

**前端变化：**
```
┌─────────────────────────┐
│ [真实视频缩略图]        │
│              ▶          │ ← 圆形播放按钮
├─────────────────────────┤
│ 视频标题（最多2行）     │
└─────────────────────────┘
```
