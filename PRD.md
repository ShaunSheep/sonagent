# 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
- 名称：sonagent（豆包AI助手）
- 类型：本地AI视频助手
- 核心功能：基于豆包大模型的智能对话助手，支持本地视频播放和语音交互

### 1.2 目标用户
- 需要本地视频管理和AI对话的用户
- 家庭用户（儿童视频为主）

### 1.3 技术栈
- 后端：Node.js + Express
- 前端：HTML5 + CSS3 + Vanilla JS
- 数据库：SQLite
- AI：豆包大模型 API

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 智能对话
- 支持文本输入和语音输入
- 基于豆包大模型的AI对话
- 支持多轮对话上下文
- 自动识别视频相关请求并搜索本地视频

#### 2.1.2 本地视频管理
- 视频文件存放在 `public/videos/` 目录
- 通过 `videos.json` 维护视频元数据
- 搜索匹配 `title` 和 `tags` 字段
- 播放路径统一走本地 `/videos/:filename`

## 4. 核心功能流程

### 4.1 视频搜索播放流程
```
用户输入 → 前端检测关键词 → 搜索本地视频
    ↓
后端匹配 videos.json → 返回 [{id, name}]
    ↓
AI 回复"找到视频了！" → 显示内嵌视频卡片
    ↓
用户点击卡片 → 请求 /api/video-path/:id
    ↓
后端返回完整信息（含 path） → 跳转播放页 → 播放
```

## v1.5 Bug修复（第二轮）

### Bug修复记录

| Bug | 描述 | 解决方案 |
|-----|------|----------|
| Bug1 | 缩略图文件名乱码 | 使用 MD5 哈希生成 16 位安全文件名 |
| Bug2 | 手机端标题显示3行 | CSS 添加固定高度 `height: 36px` |

### 技术方案

#### Bug1：安全文件名生成
```python
import hashlib

def safe_filename(video_id):
    if video_id.isascii():
        return video_id
    return hashlib.md5(video_id.encode('utf-8')).hexdigest()[:16]
```

#### Bug2：标题高度限制
```css
.video-card-title-inline {
    height: 36px;  /* 13px * 1.4 * 2 ≈ 36px */
    overflow: hidden;
}
```
